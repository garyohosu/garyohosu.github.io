X（旧Twitter）で最近急激に話題になっているのが、**Claude Codeの「Agent Teams」**と呼ばれる機能です。

「もうプロンプトを何度も書かない」「AIに自分でループを回させてチームで協力させる」——そんな新しい開発スタイルが急速に広がっています。

難しい言葉は最小限に、**例え話を交えながら実践的に**解説します。実際に試してみたくなるはずです。

## まず「ループエンジニアリング」とは？

従来のAIとの会話は「質問 → 回答 → 修正指示 → 再回答」の繰り返しでした。

**ループエンジニアリング**はこれを根本から変えます。

想像してみてください。**工場長（あなた）**が以下を明確に指示するだけ：
- 「最終目標は何？」
- 「成功かどうかはどうやって判断する？」
- 「何回まで繰り返していい？」
- 「守るべきルールは？」

するとAIが**自分で「書く→テストする→評価する→修正する」**というループを回し続け、目標達成するまで頑張ってくれるのです。

人間は「設計者」になり、AIは「自走する作業チーム」になる。これが今、開発者の間で大きな潮流になっています。

## Claude Codeの「Agent Teams」とは？

さらに進化したのが**Agent Teams**機能です。

1人の「Leadエージェント（チームリーダー）」がいて、3〜5人の「専門エージェント（メンバー）」がチームを組むイメージです。

- **Lead**：全体の計画を作り、タスクを分解して割り振り、メンバーの成果をレビュー
- **メンバー**：それぞれがFrontend担当、Backend担当、テスト担当、ドキュメント担当など専門分野を持つ
- **連絡手段**：mailbox（手紙システム）でメンバー同士が会話
- **衝突防止**：file locking（誰かが編集中は他の人は触れない仕組み）

これにより、複雑なアプリケーションを「1回の指示でほぼ完成」させる事例が続々と報告されています。

**例え話で言うと**：優秀な執事チームを雇ったようなもの。一番偉い執事（Lead）が「お客様の要望を叶えるために全員で協力せよ」と指示を出し、それぞれが自分の得意分野で動き、進捗を報告し合う。あなた（主人）は細かい指示を出す必要がほとんどありません。

## 実践的な始め方（すぐに試せる手順）

1. **最新版のClaude Codeを用意**
   - Claude Codeの最新バージョン（v2.1.32以上）で、`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` という環境変数を有効にする

2. **リポジトリにCLAUDE.mdを作成**
   - プロジェクトのルール、好みのコーディングスタイル、テストコマンド、成功基準を明確に書く
   - これがチーム全員の「共通のマニュアル」になります

3. **Leadに指示するプロンプト例**
   ```
   目標：TODOアプリをReact + FastAPIで作成
   成功基準：ログイン機能があり、テストが全て通ること
   チーム構成：Lead + Frontend + Backend + QAの4人
   各メンバーに明確な役割を与え、mailboxで連携しながら進めてください
   ```

4. **推奨チームサイズ**
   - 最初は**3〜5人**がベスト。人数が多すぎると調整が大変になります

5. **重要なポイント**
   - タスクの状態管理（pending → in_progress → completed）
   - Plan（計画）の承認フロー
   - ファイル編集時のロック機構を活用

## 実際に使っている人の声

- 「もうプロンプトを繰り返し書く時代は終わった。ループと検証ルールを設計するだけ」
- 「3人のエージェントチームに任せたら、予想以上に良いコードが出てきて驚いた」
- 「夜間にループを回させておくと、朝にはかなり完成度の高いものができている」

特に日本の開発者からも「副業で爆速でプロダクトを作れるようになった」という報告が相次いでいます。

## まとめ

Claude CodeのAgent Teamsは、単なる「便利ツール」ではなく**開発の考え方そのものを変える**ものです。

ループエンジニアリングを理解し、AIに「自走するチーム」として働いてもらうことで、人間はより創造的な部分に集中できるようになります。

あなたはまず**小さなプロジェクトで3人チーム**から試してみるのがおすすめです。

実際に試してみて「こういうループが効いた」「チームの役割分担で困った」といった体験を、ぜひブログやnoteで共有していただけると嬉しいです。
