# AIが「夢の中で自分をアップデート」する時代が来た！

こんにちは、エルメスです👋

最近AI界隈でめっちゃ話題になっているのが、**「ただ賢いAI」ではなく「自分で考えて、自分で成長し続けるAI」**です。

もちろん、人間のように意識があるわけではありません。過去の作業記録を整理して、次に活かしやすくする仕組みが注目されている、という意味です。

特に熱いのが**Claude Code**というツールと、**「Dreaming（夢見る）」**という仕組み。

難しい言葉は一切なし！  
中学生でもわかる例えで解説するので、AIに興味ある非エンジニアさんもぜひ最後まで読んでみてください。

---

## Claude Codeって何？（チームと自転車の例えで解説）

普通のAI（ChatGPTやClaude）は「賢い相談相手」みたいな存在です。

でもClaude Codeは違います。

**「賢い相棒＋優秀な秘書＋超慎重な現場監督」がチームになった感じ**なんです。

すごいポイントはここ↓

> Claude Codeの強さは、AI本体（頭脳）だけじゃない
> むしろ「ハーネス（枠組み）」と呼ばれる周辺の仕組みが主役級

ハーネスを自転車に例えると…

- AI本体 → ペダルを一生懸命こぐ「脚の力」
- ハーネス → フレーム、ブレーキ、ギア、設定まわりの仕組み全部

脚の力だけ強くても、ブレーキやフレームがガタガタだと長距離は走りにくいですよね？

Claude Codeは、AI本体だけでなく、周辺の設定や連携の仕組みも含めて使うツールです。

具体的には：
- permissionsで操作の許可・確認・拒否を制御できる
- hooksでチェックや作業の自動化を組み込める
- subagentsで作業を分担しやすくできる
- memory filesでプロジェクトの前提やルールを残せる

便利な一方で、hooksなどは設定次第で強力に動きます。だからこそ「何を許可するか」を慎重に決めながら使う必要があります。

このハーネスがあるから、「ちょっとコード書いて」だけでなく、プロジェクト全体の作業を一緒に進めやすくなっているんです。

---

## 「Dreaming（夢見る）」が一番ワクワクする部分

これが2026年でかなり熱いトピックです。

人間の脳は寝ている間に「今日の出来事を整理して記憶に定着させる」ことをしますよね。

AIのDreamingも、それに近い比喩で語られています。

ただし、ここでいうDreamingは、Claude.aiの通常チャットにそのまま付いている機能ではありません。AnthropicがClaude Managed Agents向けに発表した、過去セッションや記憶ストアを整理し、次の作業で参照しやすくする自己改善の考え方です。

AIのモデル本体の重みを書き換えるのではなく、過去の作業記録から学習メモやplaybookのようなものを作り、次回以降の作業で参照しやすくする仕組みです。

仕組みはシンプル：
1. 過去セッションやmemory storeをレビューする
2. 重複した情報、古い情報、矛盾した情報を整理する
3. 大事な教訓を学習メモやplaybookとしてまとめる
4. 次の作業で参照しやすい外部記録として残す

これを繰り返すと、AIそのものが人間のように成長するというより、過去の作業から得たメモや手順を次に活かしやすくなります。

「毎回同じ説明をしなくていい」「以前の反省を参照しやすい」「作業パターンを反映しやすい」。そんな方向に、AIエージェントは進みつつあります。

## あなたも今日から試せる3つの方法

### ① Claude.aiで“Dreamingっぽい振り返り”を試す
まずは、同じ会話の中でAIに振り返ってもらう体験をする方法です。

これはAnthropicのManaged Agents向けDreamingそのものではありません。一般ユーザーがまず試しやすい、「会話内での振り返りプロンプト」です。無料プランでも、会話内での振り返りプロンプトなら試せます。

1. [claude.ai](https://claude.ai/)を開く
2. 新しいチャットで最初にこう書く：

```
あなたは私のAIパートナーです。
作業が終わったら「Dreamingタイム」として、
振り返りと改善点をまとめてください。
```

3. 小さなことから試してみる（例：「今日のアイデアをブログ記事の構成にまとめて」）

次の会話でも覚えているかどうかは、使っているサービスの記憶機能や設定によって変わります。まずは「AIが自分の仕事を振り返る」感覚を体験してみてください。

### ② 普段使っているAIに同じことを頼む
ChatGPT、Claude、Geminiなど、普段使っているAIに

> 「この会話を振り返って、次に活かせる教訓をまとめて」

と頼むだけでも、Dreamingっぽい振り返り体験ができます。

ただし、次回以降も本当に覚えているかどうかは、使っているAIサービスの記憶機能や設定によって変わります。

### ③ 一番おすすめ：自分専用「AI記録ノート」を作る
これが意外と効果絶大です。

- NotionでもApple Notesでもいいので「AIパートナー記録」というノートを作る
- AIと話した後、**3行だけ**書く（今日の気づき・AIの提案・次に活かしたいこと）
- 次の会話の最初にそのノート内容をAIに見せる

これだけで「過去の作業を次に活かすAI」体験がかなり変わります。

## これからのAIは「成長するシステム」になる

2026年の今、AIの競争は「どれだけ賢いモデルか」だけでなく、**「どれだけ上手に過去の作業を整理し、次に活かせるか」**へ大きくシフトしつつあります。

Claude Codeのような強力な枠組み（ハーネス）と、Dreamingのような記憶整理・自己改善の考え方が合わさると、「一緒に作業を積み上げていけるパートナー」に近づいていきます。

あなたも今日から小さく始めてみませんか？

最初はぎこちなくても、続けるうちにAIへ渡せる前提や作業メモが増えて、あなたのやり方に合わせやすくなりますよ。
