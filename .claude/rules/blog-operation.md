# ブログ運用ルール（必ず守ること）

このファイルは作業のたびに遵守チェックを行う必須ルール。
詳細な背景・インシデント記録・コマンドは AGENTS.md を参照すること。

## 1. 作業前

- AGENTS.md を読んでから作業を始める。
- 記事作成前に `ls _posts/ | grep -i "<キーワード>"` で類似ファイル名を確認し、スラッグの衝突を避ける。

## 2. 記事作成時

- YAMLフロントマターの入れ子の `"` は `「」` に変換する。
- 数字のみのタグ・カテゴリーは必ずクォートする（例: `"403"`）。
- `image` / `image.path` に指定する画像は `assets/img/` に実在するファイルのみ使う。存在しない画像パスを書かない。
- 外部URLは有効性を確認し、すべて `https://` で始めること（`http://` はビルドエラーになる）。
- ainews 記事は manga 画像を記事と同時に生成・追加する。
- 生成した記事本文は必ず `_posts/` 配下の新規ファイルに保存する。`ainews.md` などの手順ファイルを上書きしない。

## 3. プッシュ前

- `git status` と `git diff` で意図しない変更（特に `_config.yml`）がないか確認する。
- 可能であれば `bundle exec jekyll build` でローカルビルドを確認する。
- コミットメッセージは原則英語を使う（PowerShell環境の文字化け防止）。

## 4. プッシュ後（最重要）

- 必ず `node scripts/ai-post-push-check.mjs` を実行する（人手確認に頼らない）。
- `Build and Deploy` が Success であること、サイトトップと変更記事URLが 200/3xx 応答であることを確認する。
- 失敗した場合はその場で修正し、再push後に再度確認する。

## 5. 記録

- 変更が発生したら必ず CHANGELOG.md に追記する（変更内容と理由）。
- 問題を解決したら必ず AGENTS.md の「インシデント記録」に追記し、再発防止策をチェックリストに追加する。
