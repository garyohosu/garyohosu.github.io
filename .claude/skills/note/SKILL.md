# 目的

note に公開した記事のURLを受け取り、ブログ掲載用の紹介記事を作成する。
元記事の趣旨を正確に把握した上で、読みやすく自然な紹介文にまとめる。

# 入力

- note 記事の URL（例: `https://note.com/hantani/n/xxxxxxxx`）

# 手順

1. **記事を読む** — 指定URLにアクセスし、タイトル・本文・タグ・公開日・OGサムネイルURLを取得する
2. **内容を把握する** — 記事の要点・構成・意図を理解する（要約ではなく、主張・手順・対象読者まで把握する）
3. **紹介記事を構成する** — 下記「出力構成」に従いブログ向け紹介文を作成する
4. **ファイルを出力する** — `note_intro.md` として保存する
5. **ブログ記事として追加する** — AGENTS.md のルールに従い `_posts/` へ記事ファイルを作成し、OGサムネイルを `assets/img/` へ保存する

# 出力構成

## タイトル案（3本）

元記事タイトルをベースに、ブログ読者に向けた見出し候補を3本挙げる。
煽り表現・誇張は避け、内容を正確に反映する。

## 記事の要点

- 箇条書きで3〜5点にまとめる
- 手順・主張・事実を簡潔に

## 読みどころ

- 特筆すべき視点・工夫・独自性を1〜3点で説明する
- 単なる要約でなく、なぜ読む価値があるかを示す

## こんな方におすすめ

- 想定読者を2〜3パターン挙げる

## 元記事リンク

```
[{記事タイトル}]({URL})
```

## ブログ掲載用本文（Markdown）

既存のnote紹介記事（`_posts/` 内の `note-` プレフィックス記事）のフォーマットに合わせて出力する。
フロントマター（`layout`, `title`, `date`, `categories`, `tags`, `description`, `image`）を含む完全な形式にする。

## SNS投稿文（任意）

X（Twitter）向けの短文を1本。140字以内。リンク含む。

# 品質チェック

- [ ] 元記事の趣旨を歪めていないか
- [ ] 過剰な煽り表現・誇張がないか
- [ ] 引用は最小限か（基本は要約で書く）
- [ ] フロントマターの引用符が正しいか（入れ子の `"` は `「」` に変換）
- [ ] `image.path` に指定した画像ファイルが `assets/img/` に存在するか
- [ ] 外部URLがすべて `https://` で始まっているか
- [ ] タグ・カテゴリーに数字のみの値がある場合はクォートしているか

# 注意

- ブログ記事追加時は AGENTS.md のルール（チェックリスト・CHANGELOG更新・push後確認）に従う
- OGサムネイル画像は note の `assets.st-note.com` から取得してローカル保存する
- ファイル名スラッグは `YYYY-MM-DD-note-{内容キーワード}.md` 形式にする
